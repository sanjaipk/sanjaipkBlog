Introduction:

SOAP services are web services. WSDL is created for services that are involving SOAP messages.

SOAP message structure:

SOAP message contains 4 important parts as follows:

  • Envelope :
    This is a tag that encloses all of the SOAP message.
  • Header :
    it can have some common request level properties that might be useful to process each request coming to that endpoint. E.g:- digital signature, user details to map the data to.
    Each header can have 2 attributes:
    Actor: would be the recipient of the request
    MustUnderstand: mentions if recipient can ignore the header.
  • Body :
    contains actual data that will transferred in the message. The data is represented as a XML chunk.
  • Fault :
    part of a response and has details of an error occurred in service. Details include error description, error code and its source.
  • Fault Codes (4 possibilities): :
    SOAP-ENV:VersionMismatch
    Problem with xml namespace used in request. Contact service team to get rightful namespace.
    SOAP-ENV:MustUnderstand
    Required header element is not processed by server correctly.
    SOAP-ENV:Client
    Message from client is not correctly formed.
    SOAP-ENV:Server
    Server problem and could not be processed.
SOAP in WSDL:

In WSDL, endpoint is made SOAP with its binding definition
<soap:binding style='document' transport='http://schemas.xmlsoap.org/soap/http' />
