Introduction:

Layered application is software architectural approach. In this article provides details on Layered application approach with examples.

High-level process flow:

Process flow is a representation of path that will be taken by user input until it reaches data store.

Generic layering template:

It is advised to start with generic layering structure and evolve to accept process flow.
Generic layering involves separation of screens from logic and logic from data storage.

It consists of following 3 layers:
  • UI Layer :
    contains all screens and logic required to render a web page.
  • Business Layer :
    contains code that would enable business logics, validation on the input from UI layer.
  • Data Layer :
    contains all functions that helps in saving business entities to data store.
Inside a layer:

Interactions between layers:

Communication is done using finalized contracts between involved layers. These contracts are essentially data structure used to pass information. Mostly, the contracts are formatted into XML or json (simplified XML) or Class objects.Contracts for layer communication is explained in future blogs. Please do care to have a look.
