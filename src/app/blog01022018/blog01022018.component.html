Classes Explained
Introduction:

Class is a computational template defined by translating real world entities. Person, Object or anything (that can be described) can also be translated to a class.

Creation:

Let's try to create class with a set of perceptible real word items give below: Place: Athens, New york, Moscow Event: Birth, Marriage, Tour, Party People: Gandhi, Edison, Tesla It is easy to create template of an item from its instances. Instances derives its identity from its properties and behaviour. Collecting shared property/behaviour among instances would yield our template (Class).

Representation:

Classes are represented initially as class diagram. It is scientific representation of a class prior to coding or programming. Coder would translate this diagram to program using language (C#, Python, etc..) of his choice. Generally, target Audience for this representation is coders and it helps if we give some additional information in class diagram.

These include
  • Property (Noun) :
    how a property is stored for further processing - (data structure) scope of the property
  • Operation (Verb) :
    set of operations that can give a class a behaviour.
Importance:

Everything is an object in computing. Any object is instance of some class. It can either provide data or behave. Whole computer paradigm rests on classes.
Over years scientists came up with algorithms, design patterns, programming languages and many more to convert real world entities to simple computational representation of classes.
