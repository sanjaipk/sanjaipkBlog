Introduction:

Web services are server applications that does not have user interface. Instead visual interface, a response is processed back.

A B C of Web service:

Each web service has an address (A) which is a combination of IP and port (generally aliased using to an URL). Web service would be callable with certain protocol (HTTP, TCP, etc..). Binding (B) tells client on how to connect to services. Web service also its input data structure. also known as contract Â© of web service.

WSDL and WADL:

WSDL/WADL are special XML that are used to represent the web service definitions. Definitions essentials contains ABCs of the web service

SOAP services:

SOAP is acronym for Simple Object Access Protocol. It is a messaging standard used in communication between 2 softwares. It is basically an XML sent over HTTP requests and responses. SOAP requests are HTTP POST calls made with SOAP message request body. Response would have content type as text/xml.

REST services:

Rest is acronym for REpresentational State Transfer. Everything is a resource in REST and is identified using URI (Uniform Resource Identifier ). Here, CURD can be analogized to PUT, POST, GET, DELETE operations of HTTP. Also, PUT and DELETE are called Idempotent Operations as it does same thing whenever called.

Microservices:

A new style of web services. It is basically building applications as suites of services. Services here would be independently deployed and can be managed by different teams. Componentization via services and designing for failures are key approaches to enable microservices
